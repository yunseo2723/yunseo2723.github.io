  • 카카오맵 API를 활용한 지도 기반 요양원 선택 UX
  • 요양원별 서비스 항목을 반영한 월 예상 비용 계산기 제공

  • PostgreSQL 기반으로 요양원(약 3만 건), 리뷰, 게시물, 댓글 데이터 모델링
  • Python 스크립트를 활용해 테스트용 사용자·리뷰 데이터 생성
  • Airflow + FastAPI로 구성된 분석 서버에서 리뷰 데이터를 1시간 단위로 정기 분석

Team

프로젝트 개요

사건, 오늘은 네이버 뉴스에서 범죄 관련 기사를 수집해 키워드 단위로 분류·노출하고, 사용자 반응 데이터를 기반으로 실시간 인기 기사를 제공하는 서비스입니다. 기사에 작성된 댓글을 분석해 AI 기반 감성 분석 결과를 시각화하고, 조회수·댓글·좋아요 데이터를 결합해 이슈 흐름을 직관적으로 파악할 수 있도록 설계했습니다.

핵심 기능 및 서비스 흐름

  • 네이버 뉴스 크롤링 기반 기사 수집 및 키워드 분류
  • 기사별 댓글 작성 및 좋아요 기능
  • OpenAI API를 활용한 댓글 감성 분석(긍·부정)
  • Redis 캐싱을 통한 기사 조회 성능 개선
  • Kafka 기반 이벤트 집계로 실시간 인기 TOP10 기사 노출

담당 역할

  • 풀스택 개발
  • 로그인/기사/댓글/인기 기사 등 API 구현
  • 댓글 AI 감성 분석 기능 구현, 분석 결과를 저장하고 그래프로 시각화

기술 스택 및 아키텍처

  • Architecture: Spring 기반 MSA 구조
  • Backend: Spring Boot
  • Frontend: React
  • Service Discovery: Netflix Eureka
  • API Gateway: Spring Cloud Gateway
  • Cache: Redis
  • Message / Event: RabbitMQ, Kafka
  • Infra & Deploy: Netlify, AWS EC2 (Docker)

성과 및 배운 점

  • MSA 환경에서 서비스 디스커버리, 메시지 큐, 캐싱을 활용한 확장 구조 경험
  • 동기 처리의 한계를 인식하고 비동기·이벤트 기반 아키텍처 설계 경험
  • AI(OpenAI)를 서비스 기능에 자연스럽게 통합한 경험

우파루 오딧세이 조합법 – 게임 조합 정보 제공 Android 앱

Solo

프로젝트 개요

우파루 오딧세이 조합법 앱은 모바일 게임 우파루 오딧세이에서 복잡하게 흩어져 있는 캐릭터(우파루) 조합 정보를 체계적으로 정리해 사용자가 원하는 결과를 빠르게 찾을 수 있도록 만든 조합 정보 앱입니다. 단순 정보 나열이 아니라, 조합 규칙과 조건을 구조화해 검색·필터·추천 흐름으로 제공하는 것을 목표로 했습니다.

서비스 운영현황

  • Google Play Store 정식 배포
  • 누적 사용자 약 3만 명
  • 사용자 피드백을 반영해 지속적인 기능 개선 및 업데이트 진행 중

핵심 기능 및 서비스 흐름

  • 부모 우파루 조합을 기반으로 한 결과 우파루 조회
  • 원하는 우파루를 기준으로 한 역방향 조합 검색
  • 속성·특수 조건 기반 필터 및 정렬 기능
  • 조합 결과를 빠르게 탐색할 수 있는 리스트 중심 UX

데이터 구조 & 알고리즘

  • 우파루, 속성, 조합 규칙을 정규화된 데이터 구조로 설계
  • 조합 연산 시 불필요한 계산을 줄이기 위한 조건 분기 및 사전 필터링
  • 앱 실행 시 필요한 데이터를 미리 로딩해 검색 응답 속도와 UX 개선

기술 스택 및 아키텍처

  • Platform: Android
  • Language: Kotlin
  • Architecture: 로컬 데이터 기반 단일 앱 구조
  • Data Handling: 정규화된 조합 데이터 구조
  • UI: RecyclerView 기반 리스트 UI
  • Deploy: Google Play Store

성과 및 배운 점

  • 게임 도메인의 규칙을 분석해 데이터 구조와 알고리즘으로 문제를 해결하는 경험
  • 단순 CRUD가 아닌 조합 계산·검색 성능을 고려한 로직 설계 경험
  • 사용자 입장에서 “빠르게 원하는 정보에 도달하는 UX”의 중요성을 체감
